# MLOps Stress Testing — Income Prediction

An end-to-end MLOps pipeline that trains, evaluates, and stress-tests multiple classification models on the [UCI Adult Income dataset](https://archive.ics.uci.edu/ml/datasets/adult). The best model is automatically selected and served via a FastAPI endpoint.

---

## Project Structure

```
mlops-stress-testing-main/
├── data/
│   ├── adult.csv              # Raw dataset
│   ├── X_test.csv             # Test features (generated by train.py)
│   ├── y_test.csv             # Test labels (generated by train.py)
│   └── simulated_data.csv     # Drift simulation data
├── models/
│   ├── best_model.json        # Name of the selected best model
│   ├── RandomForest.pkl
│   ├── LogisticRegression.pkl
│   ├── DecisionTree.pkl
│   ├── KNN.pkl
│   └── SVM.pkl
├── tests/
│   ├── bias_test.py           # Demographic parity check
│   ├── drift_test.py          # PSI-based data drift detection
│   ├── recall_test.py         # Minimum recall threshold check
│   └── robustness_test.py     # Prediction stability under input perturbation
├── app.py                     # FastAPI inference server
├── train.py                   # Model training + best model selection
├── simulate_data.py           # Generates drifted data for drift testing
├── Dockerfile
└── requirements.txt
```

## Usage

### 1. Train models

Trains five classifiers, logs metrics to MLflow, and saves the best model based on a combined score of recall, fairness, and robustness.

```bash
python train.py
```

### 2. Simulate data drift

Generates a shifted version of the test data to use in drift detection.

```bash
python simulate_data.py
```

### 3. Run stress tests

```bash
python tests/recall_test.py
python tests/bias_test.py
python tests/robustness_test.py
python tests/drift_test.py
```

### 4. Start the API

```bash
uvicorn app:app --reload
```

Then POST to `http://localhost:8000/predict` with a JSON body:

```json
{
  "age": 39, "workclass": 4, "fnlwgt": 77516, "education": 9,
  "education_num": 13, "marital_status": 4, "occupation": 1,
  "relationship": 1, "race": 4, "sex": 1,
  "capital_gain": 2174, "capital_loss": 0,
  "hours_per_week": 40, "native_country": 39
}
```

---

## Stress Tests

| Test | What it checks | Pass condition |
|---|---|---|
| Recall | Model recall on test set | recall ≥ 0.5 |
| Bias | Demographic parity between male/female predictions | difference ≤ 0.2 |
| Robustness | Prediction flip rate when `hours.per.week` is nudged by +1 | flip rate ≤ 15% |
| Drift | PSI on `hours.per.week`, `age`, `capital.gain` vs simulated data | PSI ≤ 0.25 |

---

## Model Selection

After training, each model is scored on a combined metric:

```
combined_score = (recall + (1 - bias) + (1 - flip_rate)) / 3
```

The model with the highest combined score is saved to `models/best_model.json` and loaded by the API.
